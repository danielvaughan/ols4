# V1SearchController Implementation Spec

## Summary

- Problem: `parseDocumentJson(...)` uses a local variable named `_json`, which does not follow the project naming pattern and is flagged by Sonar under `java:S117`.
- Goal: Rename that local variable to a conventional descriptive name such as `jsonPayload` while preserving the existing control flow and JSON parsing behavior.
- In scope: Rename the local variable and update its local references within `parseDocumentJson(...)`.
- Out of scope: Changing the exception type, altering JSON parsing behavior, or refactoring adjacent transformation calls.

## Related Specs

- `SG-IML-00052`: Adjacent fix for the generic exception thrown when the same payload is missing.

## Existing Code Touchpoints

- `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1SearchController.java`: `parseDocumentJson(...)` reads `JSON_PAYLOAD_FIELD` from the Solr document into the `_json` local variable and then parses it.
- Existing `JSON_PAYLOAD_FIELD` constant: suggests the preferred local naming direction should be `jsonPayload` or an equivalently descriptive camelCase name.
