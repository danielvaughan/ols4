# V1OntologyTermController Implementation Spec

## Summary

- Problem: `V1OntologyTermController` repeats the exact `"UTF-8"` literal in `14` `UriUtils.decode(...)` calls, which triggers Sonar rule `java:S1192`.
- Goal: Replace those duplicated charset literals with a single controller-scoped constant derived from `StandardCharsets.UTF_8.name()` without changing request handling or response payloads.
- In scope: Add one private class-level constant named `URI_DECODE_CHARSET`, import `java.nio.charset.StandardCharsets`, and replace every flagged `UriUtils.decode(..., "UTF-8")` argument in `V1OntologyTermController` with that constant.
- Out of scope: Switching to a different decode overload, changing endpoint mappings, changing repository/service calls, or refactoring unrelated controller concerns.

## Library And Framework Decisions

- Reuse:
  - `java.nio.charset.StandardCharsets`: preferred over a hardcoded charset string because it is part of the JDK and provides the canonical UTF-8 charset name.
  - Controller-local `private static final String` constant: keeps the Sonar fix local to the flagged class.
- New dependency:
  - `None`: no new library is justified for a local charset literal extraction.
- Rejected options:
  - Repeating `StandardCharsets.UTF_8.name()` inline at each decode site: rejected because it still duplicates the decode charset expression across multiple methods.
  - Introducing a shared utility class just for this charset name: rejected because it widens scope beyond the single controller backlog item.
  - Changing to a different `UriUtils.decode` overload: rejected because the current code already uses the string-based contract and the backlog item is not a framework migration.

## Functional Requirements

- `FR-1`: `V1OntologyTermController` shall declare exactly one private class-level constant named `URI_DECODE_CHARSET` with value derived from `StandardCharsets.UTF_8.name()`.
- `FR-2`: Every in-scope `UriUtils.decode(..., "UTF-8")` call in `V1OntologyTermController` shall be replaced with `UriUtils.decode(..., URI_DECODE_CHARSET)`.
- `FR-3`: No standalone `"UTF-8"` literal shall remain in `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1OntologyTermController.java` after the refactor.
- `FR-4`: Request mappings, exception behavior, repository lookups, and response serialization shall remain unchanged.
- `FR-5`: New imports required for the refactor shall remain grouped at the top of the file using the repo's standard import order.
