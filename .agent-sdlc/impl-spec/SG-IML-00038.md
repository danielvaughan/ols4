- Problem: `V1SelectController` repeats the exact `ontology_prefix` literal in `3` places inside the default `fieldList` population and `outDoc` response mapping branches inside `select(...)`, which triggers Sonar rule `java:S1192`.
- Goal: Replace those duplicated literal uses with a single controller-scoped constant named `ONTOLOGY_PREFIX_FIELD` without changing the public contract or internal runtime semantics.
