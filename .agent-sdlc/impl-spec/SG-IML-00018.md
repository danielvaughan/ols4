# V1SearchController `broad_synonyms` Constant Extraction Spec

## Summary

- Problem: `V1SearchController` repeats the `broad_synonyms` response-field key across default field selection and output mapping, producing the duplication tracked by `SG-BLG-00026`.
- Goal: Replace the repeated response-field key with one private constant while preserving the outward `broad_synonyms` field name.
- In scope: The internal `broad_synonyms` key usage in `V1SearchController`.
- Out of scope: Renaming the API field, changing the synonym source property, or altering other response-field keys.

## Related Specs

- `Sibling synonym-field constant extractions in the same controller must preserve the current outward field names.`
