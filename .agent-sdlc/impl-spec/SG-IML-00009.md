# V1SearchController `score` Constant Extraction Spec

## Summary

- Problem: `V1SearchController` repeats the literal `score` across field selection, sorting, and response output mapping, producing the duplication tracked by `SG-BLG-00017`.
- Goal: Replace the repeated field name with one private constant while keeping the `score` request/output behavior unchanged.
- In scope: The internal `score` field-name literal in `V1SearchController`.
- Out of scope: Changing whether `score` can be requested, changing the sort order, or changing the response payload key.

## Related Specs

- `Sibling response-field constant extractions in V1SearchController are tracked separately and must preserve the same response keys.`
