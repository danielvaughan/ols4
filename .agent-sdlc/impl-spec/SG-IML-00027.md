# V1SearchController Implementation Spec

## Summary

- Problem: `V1SearchController` repeats the exact `ontology_name` literal in `3` places inside the default `fieldList` population and `outDoc` response mapping branches inside `search(...)`, which triggers Sonar rule `java:S1192`.
- Goal: Replace those duplicated literal uses with a single controller-scoped constant named `ONTOLOGY_NAME_FIELD` without changing the public contract or internal runtime semantics.
- In scope: Add `ONTOLOGY_NAME_FIELD` as a private class-level constant and replace the exact standalone `ontology_name` occurrences described by Sonar in `V1SearchController`.
- Out of scope: changing request mappings, renaming public fields or parameters, or merging this change with unrelated refactors, reverting or reformatting the current in-progress `V1SearchController` worktree changes, including the existing `SHORT_FORM_FIELD` extraction.
