# Use shared short_form constant in V1SelectController

## Summary

- Problem: `V1SelectController` repeats the `"short_form"` literal in default field-list population and response mapping, which triggers Sonar `java:S1192`.
- Goal: Replace every targeted `"short_form"` occurrence with the existing shared constant from `V1NodePropertyNameConstants` and preserve the serialized key text.
- In scope:
  - Update only the `"short_form"` occurrences targeted by this ticket in `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1SelectController.java`.
  - Import and reuse `V1NodePropertyNameConstants.SHORT_FORM` rather than inventing a new constant.
  - Keep `/api/select` response semantics and default-field order unchanged.
- Out of scope:
  - Refactoring unrelated field-name literals beyond the sibling tickets.
  - Changing the response key text.
  - Touching other controllers.
