# V2LLMController Implementation Spec

## Summary

- Problem: `V2LLMController` repeats the exact `model` literal in `3` places inside the `getLLMModels()` response assembly and sort logic, which triggers Sonar rule `java:S1192`.
- Goal: Replace those duplicated literal uses with a single controller-scoped constant named `MODEL_KEY` without changing the public contract or internal runtime semantics.
- In scope: Add `MODEL_KEY` as a private class-level constant and replace the exact standalone `model` occurrences described by Sonar in `V2LLMController`.
- Out of scope: changing request mappings, renaming public fields or parameters, or merging this change with unrelated refactors.

## Library And Framework Decisions

- Reuse:
  - Controller-local `private static final String` constant: smallest maintainable fix for Sonar `java:S1192` in a single controller.
  - Existing controller flow and public field names: must stay unchanged after the constant extraction.
- New dependency:
  - `None`: no new library is required for a local literal extraction.
- Rejected options:
  - Renaming the public field or request parameter `model`: rejected because the backlog item is only about internal literal deduplication, not contract changes.
  - Folding several neighboring literal extractions into one broad refactor: rejected because each backlog item must remain traceable and independently implementable.
  - Reusing a shared cross-controller constants class: rejected because the requested fix is intentionally controller-local.

## Functional Requirements

- `FR-1`: `V2LLMController` shall declare exactly one private class-level constant named `MODEL_KEY` whose value is exactly `model`.
- `FR-2`: Use `MODEL_KEY` for the response-map key in `getLLMModels()` and its sort comparator while leaving all request-parameter names unchanged.
- `FR-3`: No exact standalone `model` literal flagged by this backlog item shall remain in the targeted branch(es) after the refactor.
- `FR-4`: Request mappings, response body shape, Solr parameter semantics, and repository/service interactions shall remain unchanged.
- `FR-5`: If `MODEL_KEY` or an equivalent constant already exists in the controller by the time implementation begins, the implementation shall reuse that existing constant instead of creating a duplicate.
