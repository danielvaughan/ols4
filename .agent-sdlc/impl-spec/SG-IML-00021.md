# Replace UTF-8 URI decoding literals in V1OntologyPropertyController

## Summary

- Problem: `V1OntologyPropertyController` repeats the `"UTF-8"` literal across seven `UriUtils.decode` call sites, which triggers Sonar `java:S1192`.
- Goal: Replace every decode literal with `StandardCharsets.UTF_8.name()` and keep existing route behavior intact.
- In scope:
  - Update all `UriUtils.decode(..., "UTF-8")` call sites in `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1OntologyPropertyController.java`.
  - Add the `StandardCharsets` import if needed.
  - Keep route mappings, repository access, and jsTree behavior unchanged.
- Out of scope:
  - Changes to any other controller.
  - Shared helpers or refactors outside this file.
  - Response payload and status changes.
