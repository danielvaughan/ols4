# V1SearchController Implementation Spec

## Summary

- Problem: `V1SearchController.addOntologyFilter(...)` currently uses `stream().collect(Collectors.toList())` to lowercase ontology ids before joining them, which triggers Sonar rule `java:S6204`.
- Goal: Replace that collection step with `stream().toList()` while preserving the resulting lowercase ontology-id order and filter-query string.
- In scope: Update only the lowercasing stream pipeline in `addOntologyFilter(...)`.
- Out of scope: Changing ontology validation, changing filter-query formatting, or refactoring unrelated stream usage elsewhere in the controller.

## Library And Framework Decisions

- Reuse:
  - JDK `Stream.toList()`: direct Sonar-compliant replacement for the current lowercased ontology-id list construction.
  - Existing validation and filter-query assembly logic: keep the same lowercase list contents and join behavior.
- New dependency:
  - `None`
- Rejected options:
  - Refactoring multiple stream pipelines in one change: rejected because this backlog item is scoped to one flagged path.
  - Changing the filter query to avoid list materialization entirely: rejected because the current logic is already correct and the issue is only about the collection idiom.

## Functional Requirements

- `FR-1`: The lowercasing stream pipeline in `addOntologyFilter(...)` shall use `stream().toList()` instead of `stream().collect(Collectors.toList())`.
- `FR-2`: The resulting `lowercasedOntologies` values and ordering shall remain unchanged.
- `FR-3`: Ontology validation and the final `ontologyId: (...)` filter-query text shall remain unchanged.
