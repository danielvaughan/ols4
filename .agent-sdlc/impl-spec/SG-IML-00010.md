# V1SearchController `_json` Constant Extraction Spec

## Summary

- Problem: `V1SearchController` repeats the raw `_json` field name when requesting Solr fields, reading the document payload, and describing the fail-fast error, producing the duplication tracked by `SG-BLG-00018`.
- Goal: Replace the repeated `_json` field-name literal with one private constant and keep the same fail-fast behavior if that field is missing.
- In scope: The internal `_json` field-name literal in `V1SearchController`.
- Out of scope: Changing the raw Solr payload field, relaxing the null check, or changing the response body shape.

## Related Specs

- `Sibling field-name constant extractions in V1SearchController must preserve the same Solr and response keys.`
