# V1SearchController `isLeaf` Parameter Implementation Spec

## Summary

- Problem: `isLeaf` is declared as `@RequestParam(required = false) boolean isLeaf`, which makes absence implicit and is flagged by Sonar for using a primitive on an optional query parameter.
- Goal: Convert `isLeaf` to `Boolean` and normalize absence explicitly to preserve today's "no leaf-only filter unless true" behavior.
- In scope: The `isLeaf` parameter declaration and the `hasChildren:false` filter branch in `search`.
- Out of scope: Other optional parameter fixes, broader search refactoring, or output field changes.

## Library And Framework Decisions

- Reuse:
  - `Spring MVC wrapper binding`: `Boolean` is the correct optional representation for a boolean query parameter.
- New dependency:
  - `None`: the ticket is controller-local and structural.
- Rejected options:
  - `Using Object`: removes type safety for a boolean query parameter.
  - `Changing the leaf filter expression`: would widen scope beyond the signature fix.

## Functional Requirements

- `FR-1`: Change the `isLeaf` request parameter type from primitive `boolean` to `Boolean`.
- `FR-2`: Normalize the wrapper once to a local primitive flag such as `Boolean.TRUE.equals(isLeaf)`.
- `FR-3`: Preserve current behavior where only `isLeaf=true` adds `hasChildren:false`, while omitted and false values leave that filter absent.
