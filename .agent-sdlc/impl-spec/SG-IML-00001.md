# V1SearchController Implementation Spec

## Summary

- Problem: `V1SearchController.search` repeats the exact literal `"short_form"` in its default field population and response mapping logic, which triggers SonarQube rule `java:S1192`.
- Goal: Replace those exact uses with a single controller-scoped constant without changing the `/api/search` contract or response payload shape.
- In scope: Add one class-level constant in `V1SearchController` and swap the exact literal uses in the default `fieldList` branch and the `outDoc` mapping branch.
- Out of scope: Renaming the public `short_form` response field, changing query parameter documentation, extracting unrelated literals, or changing Solr query behavior.

## Existing Code Touchpoints

- `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/SolrFieldMapper.java`: Preserves the legacy `short_form` field mapping contract; this spec depends on leaving that contract unchanged even though no edit is planned there.

## Files To Update

- `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1SearchController.java`: Introduce a controller-local constant for the legacy response field key and replace the exact literal uses in the `search` method.

## Library And Framework Decisions

- Reuse:
  - Java `private static final String` constant: matches the class' current pattern for controller-local constants and avoids unnecessary abstraction for a single-file maintainability fix.
  - Existing `/api/search` response contract: keeps the externally visible `short_form` field name unchanged.
- New dependency:
  - `None`: no additional library is justified for a local string-literal extraction.
- Rejected options:
  - Shared constant in another module: rejected because the issue scope is local to `V1SearchController` and the legacy response alias is not currently managed as a shared domain constant.
  - Renaming `short_form` to `shortForm`: rejected because it would change the public API contract instead of fixing a maintainability issue.

## Functional Requirements

- `FR-1`: `V1SearchController` shall define a single controller-scoped constant whose value is exactly `"short_form"`.
- `FR-2`: The default `fieldList` population path shall continue adding the legacy `short_form` field when the caller does not supply any requested fields.
- `FR-3`: The response mapping path shall continue writing `JsonHelper.getString(json, "shortForm")` under the public key `short_form` when that field is requested.
- `FR-4`: The change shall not modify request parameter names, OpenAPI annotation text, Solr query-field lists, or any response key other than replacing internal exact literal duplication with constant reuse.
