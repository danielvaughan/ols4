# Replace UTF-8 URI decoding literals in V1OntologyTermController

## Summary

- Problem: `V1OntologyTermController` duplicates the `"UTF-8"` literal across fourteen `UriUtils.decode` call sites, which triggers Sonar `java:S1192`.
- Goal: Replace all decode literals with `StandardCharsets.UTF_8.name()` while preserving the current term, tree, graph, and relation route behavior.
- In scope:
  - Update every `UriUtils.decode(..., "UTF-8")` call in `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1OntologyTermController.java`.
  - Add the `StandardCharsets` import if required.
  - Leave route mappings, repository usage, graph lookup logic, and exception behavior unchanged.
- Out of scope:
  - Any controller outside `V1OntologyTermController`.
  - Shared decode helpers or broader controller cleanup.
  - Payload or status-code changes.
