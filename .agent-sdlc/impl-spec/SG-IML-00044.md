# V1SearchController Implementation Spec

## Summary

- Problem: `V1SearchController.search(...)` eagerly calls `solrQuery.toQueryString()` inside `logger.debug(...)`, which triggers Sonar rule `java:S2629` because the string is built even when debug logging is disabled.
- Goal: Preserve the current debug log message while making query-string generation conditional on debug logging being enabled.
- In scope: Update only the debug log path around `logger.debug("V1 SEARCH QUERY: {}", ...)` in `search(...)`.
- Out of scope: Changing log levels, changing the log message text, removing the debug log entirely, or altering Solr query behavior.

- Reuse:
  - Existing SLF4J `logger` field and debug message text: keeps operational output unchanged.
  - `logger.isDebugEnabled()`: smallest controller-local fix that avoids eager message construction.
- New dependency:

  - Switching to a different logging framework API: rejected because the repo already uses SLF4J.
