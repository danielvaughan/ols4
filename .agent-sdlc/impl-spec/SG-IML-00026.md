# Reduce V1SelectController select() cognitive complexity

## Summary

- Problem: `V1SelectController.select()` currently owns query normalization, Solr filter assembly, default field handling, per-document mapping, highlighting remapping, and raw response writing in one method, which pushes Sonar cognitive complexity to `48`.
- Goal: Reduce method complexity to `15` or lower by extracting focused private helpers while preserving the `/api/select` request and response contract exactly.
- In scope:
  - Refactor `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v1/V1SelectController.java` only.
  - Extract private helpers for query preparation, filter query application, default field resolution, output document mapping, highlighting remapping, and response assembly/writing.
  - Preserve the current deterministic sort, highlighting behavior, and response shape.
- Out of scope:
  - Moving logic into a new service or shared utility outside this controller.
  - Changing query parameters, response field names, or Solr query semantics.
  - Refactoring other controllers.
