# Extract model response-key constant in V2LLMController

## Summary

- Problem: `V2LLMController.getLLMModels()` repeats the `"model"` literal in map population and sort logic, which triggers Sonar `java:S1192`.
- Goal: Replace the repeated literal with a controller-local constant while preserving the `/api/v2/llm_models` response contract exactly.
- In scope:
  - Introduce a local constant in `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v2/V2LLMController.java` for the response key used by `getLLMModels()`.
  - Replace the targeted `"model"` occurrences in that method and its sort comparator.
  - Keep the response payload keys and ordering unchanged.
- Out of scope:
  - Refactoring `"model"` usage in other classes such as MCP or helper code.
  - Changing request parameter names or default model values.
  - Any controller outside `V2LLMController`.
