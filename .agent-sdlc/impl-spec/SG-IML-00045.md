# V1SearchController Implementation Spec

## Summary

- Problem: `V1SearchController.configureExplicitQueryFields(...)` currently uses `stream().collect(Collectors.toList())` when building the exact-query field suffix list, which triggers Sonar rule `java:S6204`.
- Goal: Replace that collection step with `stream().toList()` while keeping the mapped field values and downstream Solr behavior unchanged.
- In scope: Update only the `queryFields.stream().map(...).collect(Collectors.toList())` path in `configureExplicitQueryFields(...)`.
- Out of scope: Changing exact-query semantics, changing array construction, or refactoring unrelated stream usage elsewhere in the controller.
