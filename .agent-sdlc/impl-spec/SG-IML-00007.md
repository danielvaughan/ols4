# V1SearchController `edismax` Constant Extraction Spec

## Summary

- Problem: `V1SearchController` repeats the literal `edismax` in multiple `solrQuery.set(...)` calls, creating the duplication tracked by `SG-BLG-00015`.
- Goal: Replace the repeated parser value with one private constant and keep every query branch behaviorally identical.
- In scope: The `edismax` value inside `V1SearchController` only.
- Out of scope: Changing the effective parser value, changing the adjacent `defType` key, or altering any search semantics.

  - `Existing Spring MVC controller and SolrJ query construction`: the change stays local to the current controller logic.
