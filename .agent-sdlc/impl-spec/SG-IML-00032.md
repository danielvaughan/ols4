# Replace UTF-8 URI decoding literals in V2ClassController

## Summary

- Problem: `V2ClassController` repeats the `"UTF-8"` literal across nine `UriUtils.decode` call sites, which triggers Sonar `java:S1192`.
- Goal: Replace every decode literal with `StandardCharsets.UTF_8.name()` and keep existing v2 class endpoint behavior intact.
- In scope:
  - Update all `UriUtils.decode(..., "UTF-8")` call sites in `backend/src/main/java/uk/ac/ebi/spot/ols/controller/api/v2/V2ClassController.java`.
  - Add the `StandardCharsets` import if needed.
  - Preserve endpoint mappings, repository lookups, and error behavior.
- Out of scope:
  - Changes to any other controller.
  - Shared decode helpers or broader refactors.
  - Response payload or status-code changes.
